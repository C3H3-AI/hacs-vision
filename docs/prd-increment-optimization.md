# HACS Enhanced 增量 PRD — 功能与界面优化

> 版本: 1.0 | 日期: 2025-07 | 作者: 产品经理 Alice

---

## 1. 项目信息

- **项目名称**: `hacs_enhanced`
- **技术栈**: Lit (Web Components) + HA Panel API + Python (Home Assistant Custom Integration)
- **原始需求**: 在已完成安全审计与 Bug 修复的基础上，对 HACS Enhanced 面板进行 P0/P1 级别的功能增强和交互优化，提升操作效率和用户体验。

---

## 2. 产品定义

### 2.1 产品目标

1. **操作效率** — 通过键盘快捷键和快捷入口减少重复点击步骤，让高频操作（搜索、安装、切换视图）可在 1-2 次按键内完成。
2. **状态透明** — 用户在任何操作过程中（安装/更新/网络异常）都能实时感知当前状态，消除"点了按钮不知道在干什么"的黑洞体验。
3. **信息密度** — 在不增加页面跳转的前提下，将分类统计、Changelog 摘要等关键决策信息直接呈现，降低用户的认知负荷。

### 2.2 用户故事

| # | 用户故事 |
|---|---------|
| US1 | 作为 HA 管理员，我想按 Esc 关闭详情弹窗、按 Ctrl+K 直接聚焦搜索框、按数字键 1-6 快速切换 Tab，这样我可以不用鼠标高效操作面板。 |
| US2 | 作为 HA 管理员，当网络断开时我想立刻看到全局提示条告知"网络已断开"，并在网络恢复后自动重试失败的请求，这样我不会误以为操作成功了。 |
| US3 | 作为 HA 管理员，我想在安装/更新按钮上看到明确的状态流转（下载中 → 完成），这样我能知道操作是否正在进行。 |
| US4 | 作为 HA 管理员，我想通过 Header 右侧的 "+" 按钮粘贴一个 GitHub 仓库 URL 并自动检测分类后直接安装，而不需要先去 Browse 页搜索再安装。 |
| US5 | 作为 HA 管理员，我想在 Browse 页的分类标签旁看到每个分类的仓库数量，这样我可以在点击前就知道哪个分类内容最多。 |
| US6 | 作为 HA 管理员，我想在 Updates 页的卡片中直接看到最新版本的更新内容摘要，这样我可以在不跳转 GitHub 的情况下判断是否值得更新。 |

## 3. 需求池

### P0 — 必须做

#### P0-1: 键盘快捷键

**描述**: 为面板添加全局键盘快捷键支持，覆盖最常用的交互操作。

**验收标准**:
- 按 `Esc` 键关闭已打开的详情模态框（等价于点击 ✕ 按钮）
- 按 `Ctrl+K`（Mac 上 `Cmd+K`）聚焦当前视图的搜索输入框
- 按数字键 `1`-`6` 切换到对应的 Tab（1=浏览, 2=收藏, 3=已安装, 4=更新, 5=配置, 6=备份）
- 数字键快捷键仅在搜索输入框未获得焦点时生效，避免与用户输入冲突
- 快捷键在模态框打开时不触发 Tab 切换（Esc 优先关闭模态框）

**UI 交互流程**:
1. 用户在面板任意位置按下 `Ctrl+K` → 搜索框获得焦点，光标闪烁，用户可直接输入关键词
2. 用户按下数字键 `3` → 自动切换到"已安装"Tab，页面内容平滑切换
3. 用户点击卡片打开详情模态框 → 按 `Esc` → 模态框关闭，焦点回到主面板
4. 用户在搜索框输入文字时按 `2` → 正常输入字符"2"，不触发 Tab 切换

**涉及文件**:
- `frontend_src/src/hacs-enhanced-panel.js` — 添加全局 `keydown` 事件监听，处理快捷键分发
- `frontend_src/src/views/browse.js` — 提供 `focusSearch()` 公共方法
- `frontend_src/src/views/updates.js` — 提供 `focusSearch()` 公共方法
- `frontend_src/src/views/installed.js` — 提供 `focusSearch()` 公共方法
- `frontend_src/src/i18n.js` — 新增快捷键相关提示文案

#### P0-2: 网络状态感知

**描述**: 检测 API 请求失败时的网络状态，显示全局断网提示条，网络恢复后自动重试。

**验收标准**:
- 当任意 API 请求失败且 `navigator.onLine === false` 时，在页面顶部显示橙色/红色全局提示条，文案如"网络已断开，部分功能不可用"
- 当网络恢复（`online` 事件触发）时，提示条自动消失，并自动重试最近一次失败的请求
- 提示条不遮挡 Tab 栏，位于 Header 上方或 Header 内部
- 在已断网状态下，按钮操作（安装/更新等）仍可点击但请求会排队等待网络恢复
- 同一时刻只显示一个网络提示条，不重复叠加

**UI 交互流程**:
1. 用户正常操作中 → 网络断开 → 用户点击"安装" → 请求失败 → 页面顶部出现橙色提示条"网络已断开，功能暂不可用"，同时 Toast 显示"安装请求已排队，网络恢复后自动执行"
2. 网络恢复 → 提示条自动消失 → 排队的安装请求自动重试 → Toast 显示"安装已开始"
3. 用户一直在线 → 不显示任何网络提示

**涉及文件**:
- `frontend_src/src/api.js` — 在 `request()` 方法中增加网络检测逻辑、请求排队/重试机制
- `frontend_src/src/hacs-enhanced-panel.js` — 新增 `_offline` 状态属性，渲染网络提示条；监听 `online`/`offline` 事件
- `frontend_src/src/i18n.js` — 新增断网/恢复相关文案

### P1 — 应该做

#### P1-1: 进度指示器

**描述**: 安装/更新操作按钮增加明确的状态流转视觉反馈。

**验收标准**:
- 按钮状态三态流转：**默认态** → **进行中态** → **完成态**
  - 默认态：当前已有的按钮样式（如"安装"/"更新"）
  - 进行中态：按钮文字变为"⏳ 下载中..."，按钮变为 `disabled`，背景色略微变灰，显示旋转动画图标
  - 完成态：按钮文字变为"✅ 已完成"持续 2 秒，随后恢复为该仓库的最终状态（已安装/已是最新）
- 进行中态和完成态对以下位置统一生效：
  - 仓库卡片 (repo-card.js) 的操作按钮
  - 详情模态框的操作按钮
  - Updates 页的卡片更新按钮
  - Installed 页的更新按钮
- 状态流转通过响应式属性 `_installingIds` / `_updatingIds` 集合驱动，不需要轮询
- 多个仓库同时安装/更新时，各自按钮独立显示状态，互不干扰

**UI 交互流程**:
1. Browse 页 → 用户点击某仓库卡片的"安装"按钮 → 按钮文字变为"⏳ 下载中..."，按钮变灰不可再次点击
2. 安装完成 → 按钮文字变为"✅ 已完成" → 2 秒后按钮消失，卡片显示"✅ 已安装"标记和"移除"按钮
3. Updates 页 → 用户点击"立即更新" → 按钮变为"⏳ 更新中..." → 完成后变"✅ 已完成" → 2 秒后该卡片从更新列表中移除

**涉及文件**:
- `frontend_src/src/hacs-enhanced-panel.js` — 新增 `_installingIds` / `_updatingIds` 属性，提供全局操作状态追踪
- `frontend_src/src/components/repo-card.js` — 读取操作状态，渲染按钮三态
- `frontend_src/src/views/updates.js` — 卡片更新按钮增加状态流转
- `frontend_src/src/views/installed.js` — 列表项更新按钮增加状态流转
- `frontend_src/src/api.js` — install/update 方法返回操作 ID 用于状态追踪
- `frontend_src/src/i18n.js` — 新增"下载中..."/"更新中..."/"已完成"文案

#### P1-2: 快捷安装入口

**描述**: 在 Header 右侧添加 "+" 按钮，弹出浮层支持粘贴 URL 快速安装。

**验收标准**:
- Header 右侧统计数字区域旁新增圆形 "+" 按钮（视觉与现有 stat 区风格统一）
- 点击 "+" 后弹出浮层（dropdown/popover），包含：
  - URL 输入框：支持粘贴 `https://github.com/user/repo` 格式的地址
  - 分类自动检测：输入 URL 后自动从 URL 中提取仓库信息，调用后端 API 检测分类；若无法自动检测则显示分类下拉选择器
  - "安装"按钮：点击后调用安装 API
- 浮层在以下情况下关闭：点击浮层外部区域、按 Esc 键、安装成功后
- URL 输入框支持 `Ctrl+V` 粘贴，并在粘贴后自动触发分类检测
- 移动端 "+" 按钮尺寸符合 44px 触控目标，浮层全宽显示

**UI 交互流程**:
1. 用户点击 Header 右侧 "+" 按钮 → 下方弹出浮层，包含 URL 输入框（占位符："粘贴 GitHub 仓库 URL"）和分类选择器（默认"自动检测"）
2. 用户粘贴 `https://github.com/user/awesome-integration` → 输入框下方显示"检测中..." → 0.5-1s 后分类自动选中为"集成"，输入框下方显示仓库名称
3. 用户点击"安装" → 按钮变为"⏳ 安装中..."（复用 P1-1 进度指示器） → 安装成功后浮层关闭，Toast 提示"安装成功"
4. 用户粘贴无效 URL → 输入框下方显示红色提示"无效的 GitHub 仓库地址"

**涉及文件**:
- `frontend_src/src/hacs-enhanced-panel.js` — Header 区域新增 "+" 按钮和浮层组件，新增 `_showQuickInstall` 状态
- `frontend_src/src/api.js` — 新增 `detectCategory(repository)` 方法或复用 `addCustomRepo` 的分类检测逻辑
- `frontend_src/src/i18n.js` — 新增快捷安装相关文案

#### P1-3: 分类统计数字

**描述**: Browse 页的分类标签旁显示每个分类的仓库数量。

**验收标准**:
- 每个分类标签（cat-tab）右侧显示该分类下的仓库总数，格式如"集成 128"
- "全部"标签显示仓库总数
- 数字通过后端 API 返回的 `total` 和分类统计信息获取，不需要额外请求
- 若后端未返回分类统计，数字从已加载的仓库列表中计算（降级方案）
- 数字字体略小于标签文字，颜色为 `var(--secondary-text-color)`
- 分类标签宽度自适应，数字不换行

**UI 交互流程**:
1. 用户进入 Browse 页 → 分类标签加载后，每个标签旁显示数字，如"📋 全部 1842"、"🔌 集成 652"、"🧩 插件 438"
2. 用户输入搜索关键词 → 分类数字保持不变（反映全量分类统计，不受搜索过滤影响）
3. 用户点击"集成"标签 → 列表只显示集成类仓库，但数字仍为 652

**涉及文件**:
- `frontend_src/src/views/browse.js` — 新增 `categoryCounts` 属性，分类标签模板增加数字渲染
- `frontend_src/src/api.js` — `listRepositories` 响应中读取分类统计字段
- `frontend_src/src/hacs_data.py` 或 `frontend_src/src/api.py`（后端） — API 响应增加 `category_counts` 字段

#### P1-4: Changelog 预览

**描述**: Updates 页卡片中展示最新版本的更新内容摘要。

**验收标准**:
- 每个 Updates 卡片在版本号区域下方新增"更新内容"区域，显示最新版本的 Changelog/Release Notes 摘要
- 摘要来源优先级：1) GitHub Release Notes → 2) 仓库最近 commit message → 3) 显示"暂无更新说明"
- 摘要默认折叠，最多显示 3 行（约 80 字），超出部分显示"展开"链接
- 点击"展开"后显示完整内容（最多 300 字），点击"收起"折叠
- 摘要内容为纯文本，Markdown 格式仅做基础解析（列表、加粗），不做完整渲染
- 摘要通过后端 API 代理获取，前端做 1 小时本地缓存
- 若获取 Changelog 失败，不显示"更新内容"区域，不影响卡片其他功能

**UI 交互流程**:
1. 用户进入 Updates 页 → 卡片加载 → 版本号下方出现"更新内容"区域，显示摘要前 3 行
2. 用户点击"展开" → 摘要完整显示，"展开"变为"收起"
3. 用户点击"收起" → 摘要折叠回 3 行
4. 某仓库无 Release Notes → 卡片不显示"更新内容"区域，版本号和描述之间无多余空白

**涉及文件**:
- `frontend_src/src/views/updates.js` — 卡片模板新增 Changelog 区域，折叠/展开交互
- `frontend_src/src/api.js` — 新增 `getChangelog(fullName, tag)` 方法
- 后端 `api.py` — 新增 `/api/hacs_enhanced/changelog/{full_name}/{tag}` 端点
- `frontend_src/src/i18n.js` — 新增 Changelog 相关文案（"更新内容"、"展开"、"收起"、"暂无更新说明"）

- 位置：Header 上方，宽度 100%，高度约 36px
- 断网态：橙色背景 `#fff3e0`，文字"网络已断开，功能暂不可用"

## 5. 需求依赖关系

```
P0-1 键盘快捷键 ──(独立)──→ 无依赖
P0-2 网络状态感知 ──(独立)──→ 无依赖
P1-1 进度指示器 ──(被依赖)──→ P1-2 快捷安装（复用进度指示器）
P1-2 快捷安装 ──(依赖)──→ P1-1 进度指示器
P1-3 分类统计 ──(独立)──→ 无依赖
P1-4 Changelog ──(独立)──→ 无依赖
```

建议实施顺序：P0-1 → P0-2 → P1-1 → P1-3 → P1-2 → P1-4
