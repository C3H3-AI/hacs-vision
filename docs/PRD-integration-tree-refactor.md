# 📋 产品需求文档 (PRD) — 增量

## 集成管理弹窗树形重构

**版本**: v1.0 (增量)
**产品经理**: 许清楚
**日期**: 2026-06-14
**状态**: ✅ 设计决策已确认

---

## 1. 产品目标

将 HACS Vision「集成管理」模块的弹窗从**平铺条目列表 + 独立设备覆盖层**重构为**统一三级树形结构（集成 → 条目 → 设备 → 实体）**，消除设备视图的界面闪现问题，降低认知负担，提供一致的浏览和操作体验。

---

## 2. 设计决策（已确认）

| # | 决策 | 结论 | 理由 |
|---|------|:----:|------|
| D1 | 单条目集成点击卡片行为 | 统一显示条目列表 | 一致性 > 省一步点击 |
| D2 | 子项集成（AI Hub 等）展示 | 与普通条目一样展示 | 减少特殊 case，统一处理 |
| D3 | device-view.js 组件命运 | **废弃**，全部在树内完成 | 统一交互模式，消除 Flash |

## 3. 用户故事

### P0 — 核心功能（必须有）

| ID | 用户故事 | 验收标准 |
|:--:|---------|----------|
| US01 | 作为用户，点击集成卡片后看到条目列表，每条可展开/折叠显示其下的设备 | 条目行有展开箭头，点击后展开/折叠设备树 |

| US04 | 作为用户，展开/折叠设备时可看到加载状态 | 展开中显示 spinner，加载完成显示设备列表 |

## 4. UI 设计稿（文字描述）

### 4.1 树形结构总览

```
┌─────────────────────────────────────────┐
│  [Avatar] 集成名称           ⊕  ⊖  [✕]  │  ← 弹窗头部
│            N 个条目                      │
├─────────────────────────────────────────┤
│  ● 条目标题  [启用/禁用][配置][重载][✕] │  ← 条目行
│  ├─ 📦 设备名 (型号)     5个实体  [▶/▼] │  ← 设备行
│  │  ├ 💡 灯名称             开   [toggle]│  ← 实体行
│  │  ├ 🌡️ 传感器名称      26°C         │
│  │  └ 🔌 开关名称           关   [toggle]│
│  ├─ 📦 设备2              3个实体  [▼] │
│  └─ ...                                 │
│  ● 条目2       [配置][重载][✕]         │
│  └─ ...                                 │
└─────────────────────────────────────────┘
```

### 4.2 交互规则

- **条目行**：点击箭头展开/折叠设备；点击空白区域无操作（操作通过右侧按钮）
- **设备行**：点击箭头展开/折叠实体；点击设备名打开 more-info
- **实体行**：可切换实体点击执行 toggle；不可切换实体点击打开 more-info
- **状态标识**：条目左侧圆点颜色（绿/红/橙/灰）表示状态；禁用条目灰色显示

### 4.3 对比现有逻辑（增量变更）

| 当前行为 | 重构后行为 |
|---------|-----------|
| 条目行右侧有「设备」按钮，点击后 `_toggleEntry` | 条目行改为左侧展开箭头，与树形结构统一 |
| `device-view.js` 独立组件的覆盖层弹窗 | 废弃 `device-view.js`，设备数据在树内展开 |
| 设备展开后没有实体渲染 | 设备展开后渲染实体行（复用 `device-view.js` 的实体渲染逻辑） |
| 实体渲染有独立的 CSS（dv-*） | 实体渲染统一为树内样式（tree-entity） |

## 6. 成功指标

- **用户路径简化**：原来查看实体需要 3 次点击（卡片 → 条目→ 设备覆盖层）变为 2 次（卡片 → 展开条目 → 展开设备）
- **Flash 消除**：设备视图不再出现内容闪现后消失（独立覆盖层状态同步问题）
- **代码简化**：移除 `device-view.js` 独立组件，减少约 400 行代码
