## 三、关键 Bug

### 🔴 P0-1：静态文件 requires_auth=False 未生效

**现象**：`/api/hacs_vision/panel.js` 无认证访问返回 401，而 HA panel loader 的 `import()` 无法携带 Authorization header。

**日志确认**：
```
Login attempt or request with invalid authentication from ... 
Requested URL: '/api/hacs_vision/panel.js'
```

**影响**：HA 侧边栏打开 hacs_vision 面板时，Lit 组件无法加载，前端白屏。

**可能原因**：
- HA 2026.6 的中间件栈可能在外层覆盖了 `requires_auth`
- `HACSEnhancedStaticView` 的 `requires_auth = False` 被 HA 全局中间件拦截

**建议**：
```python
# Option A: 将静态文件挂到 /local/ 路径（HA 内建无需认证）
# Option B: 确保 HomeAssistantView.requires_auth = False 在新版 HA 中仍生效
```

### 🔴 P0-2：Backup Export 返回空数组

**现象**：`backup/export` 返回 `installed_repositories: [], custom_repositories: []`，但实际有 82 个已安装和 24 个自定义仓库。

**影响**：备份功能完全不可用，用户无法导出数据。

**排除**：`BackupManager.__init__` 传入 `shared_data` 是否正确初始化需检查。

### 🔴 P0-3：Search API 404

**现象**：`/api/hacs_vision/repositories/search` 返回 `{"error": "not_found"}`

**影响**：前端搜索功能不可用。

**排查**：确认 `api.py` 中的路由注册路径是否匹配。

## 四、警告

### ⚠️ W1：阻塞 I/O — `api.py:960`

```
Detected blocking call to open with args ('/config/custom_components/claw_assistant/translations/zh-Hans.json', 'r')
inside the event loop by custom integration 'hacs_vision'
```

在 `_get_translations` 方法中使用同步 `open()`，HA 2026.6 的 Python 3.14 严格检测阻塞 I/O。

**建议**：改用 `hass.async_add_executor_job()` 或 `aiofiles.open()`

### ⚠️ W2：版本号不一致

| 位置 | 版本 |
|------|------|
| `const.py` | `2.1.2` |
| `manifest.json` | `2.2.0` |
| API 返回值 | `2.1.2` |
| 仓库列表 installed_version | `v2.1.2` |
| 仓库列表 latest_version | `v2.1.2` |

manifest 声称为 2.2.0，但实际运行的是 2.1.2。HACS 更新检测可能认为已是最新。

### ⚠️ W3：多个端点未注册

以下端点返回 not_found，可能是：① 路径拼写错误 ② 未实现的功能 ③ API 路由不一致

## 八、结论

hacs_vision v2.1.2（manifest 标为 2.2.0）在 HA 2026.6.0 上的**核心数据层运行正常**（仓库查询、config entries、翻译、收藏等），但存在 **3 个 P0 阻塞问题**：

1. **前端完全不可用** — 静态文件需要认证，HA panel loader 无法加载
2. **备份功能完全失效** — 数据读取逻辑异常
3. **搜索接口不可用** — 路由问题

建议修复顺序：P0-1（前端）→ P0-3（搜索）→ P0-2（备份）→ 版本号 → 阻塞 I/O → 其他端点。
