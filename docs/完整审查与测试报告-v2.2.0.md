### 🔴 P0 — 必须修复的 Bug

#### P0-1: 版本号不一致
- **文件**: `const.py` L5
- **现象**: `VERSION = "2.1.2"` vs `manifest.json` `"version": "2.2.0"`
- **影响**: API 端点 `/api/hacs_vision/version` 返回 2.1.2，HACS 更新检测混乱
- **修复**: 统一为 `"2.2.0"`

#### P0-2: `api.py` 中 6 处 `localhost:8123` 硬编码
- **位置**: Config Flow 代理端点（L228, L246, L265, L285, L303, L320）
- **影响**: 非标准端口/反向代理/Nabu Casa 环境下 Config Flow 完全失效
- **修复**: 从 `hass.config.api.base_url` 动态获取

#### P0-3: `_install_locks` 内存泄漏
- **文件**: `hacs_operator.py` L182（install）、L249（remove）
- **现象**: `_cleanup_lock()` 只在成功路径调用；异常时锁对象永久留在字典中
- **影响**: 长期运行后字典膨胀（每个遗留锁 ~200 bytes）
- **修复**: 将 `_cleanup_lock` 移入 `finally` 块

#### P0-4: `management.js` 可能缺失 `getCategoryColor` 导入
- **文件**: `frontend_src/src/views/management.js` L1-7
- **风险**: 如果模板中直接调用 `getCategoryColor()`，运行时 ReferenceError
- **修复**: 检查并补充 `import { getCategoryColor } from '../shared/constants.js'`

### 🟡 P1 — 性能问题

#### P1-1: `get_all_repos_from_hacs()` N+1 await
- **文件**: `hacs_operator.py` L~590
- **现象**: 循环中对每个仓库 `await _find_config_entry_id(domain)`
- **影响**: 200 个仓库 × 50 个 config entries = 10,000 次比较，延迟 ~2-5 秒
- **修复**: 预构建 `domain → entry_id` 字典

```python
# 优化前
for repo in repos:
    entry_id = await self._find_config_entry_id(repo.domain)  # N次await

# 优化后
entry_map = {e.domain: e.entry_id for e in self._hacs.hass.config_entries.async_entries()}
for repo in repos:
    entry_id = entry_map.get(repo.domain)  # O(1)
```

#### P1-2: `_preloadDomainNames()` 串行加载翻译
- **文件**: `integrations-list.js` L64-110
- **现象**: `for (const domain of domains)` 中逐个 `await loadBackendTranslation()`
- **影响**: 50 个集成 = 50 次串行请求，首屏加载 ~5-10 秒
- **修复**: `Promise.allSettled()` 并行加载

#### P1-3: `_ensure_index()` 并发竞态
- **文件**: `hacs_operator.py` L~100
- **现象**: 无锁保护，多协程同时触发时会重复构建索引
- **影响**: CPU 和内存短暂翻倍，索引可能临时不一致
- **修复**: 添加 `asyncio.Lock` 保护

#### P1-4: `_filteredDomainGroups` 和 `_chipCounts` 重复计算
- **文件**: `integrations-list.js` L267-311
- **现象**: 两个 getter 每次访问都重新遍历全量 `configEntries`
- **修复**: 合并为单一计算方法，结果缓存到实例属性

#### P1-5: `api.py` 同步 I/O 阻塞事件循环
- **文件**: `api.py` L960、L997
- **现象**: `open(path, "r")` 同步读取文件，HA 日志警告
- **日志**: `Detected blocking call to open inside the event loop by custom integration 'hacs_vision'`
- **修复**: 改用 `await self.hass.async_add_executor_job()`

### 🔴 P0 Bug 实机验证

#### P0-1: 静态文件 401（**假阳性**）
- **预期**: 无认证访问静态文件应返回 200
- **实测**: `GET /api/hacs_vision/static/panel.js` (无 auth) → **HTTP 200** ✅
- **根因**: 之前测试用了错误路径 `/panel.js`，正确路径是 `/static/panel.js`
- **结论**: **不是 Bug**，无需修复

#### P0-2: Backup 空数据（**真 Bug**）
- **预期**: `GET /api/hacs_vision/backup/export` 返回 84 installed + 24 custom
- **实测**: `{"installed_repositories": [], "custom_repositories": []}` ❌
- **根因**: `backup.py` 的 `export_data()` 使用 `data.get_installed_repositories()` 读取 `.storage/hacs.data` 文件，返回的是**全量 catalog（3144 条）**而非已安装仓库
- **正确数据路径**: 应使用 `operator.get_installed_list()`（从 HACS 内存读取，正确过滤 `installed=True`）
- **修复方案**:
  ```python
  # backup.py
  async def export_data(self) -> dict:
      installed = []
      custom = []
      if self.operator and self.operator.available:
          # 从 HACS 内存读取（准确）
          installed = self.operator.get_installed_list()
          all_repos = self.operator.get_all_repos_from_hacs()
          custom = [r for r in all_repos if r.get("custom")]
      else:
          # Fallback: 文件读取
          installed = await self.data.get_installed_repositories()
          config = await self.data.get_config()
          custom = config.get("custom_repositories", [])
      return {"installed_repositories": installed, "custom_repositories": custom}
  ```

#### P0-3: Search API 404（**假阳性**）
- **预期**: 搜索功能应正常工作
- **实测**: `GET /api/hacs_vision/repositories/search?q=xiaomi` → **404 not_found** ❌
- **根因**: 搜索端点路径是 `GET /api/hacs_vision/repositories?search=xiaomi`（query 参数），不是 `/repositories/search`
- **前端实际调用**: `api.js` L~120 使用 `fetchRepositories({ search: "xiaomi" })` → 正确构造 `?search=xiaomi`
- **结论**: **不是 Bug**，测试用了错误路径

#### 修复 1: Backup 数据源切换（P0）

**文件**: `backup.py` + `__init__.py`

**当前问题**:
```python

async def export_data(self) -> dict:
    installed = await self.data.get_installed_repositories()  # 返回 3144 条全量数据
    config = await self.data.get_config()
```

**修复后**:
```python
# backup.py
class BackupManager:
    def __init__(self, hass, shared_data=None, operator=None):
        self.data = shared_data or HACSData(hass)
        self.operator = operator  # 新增
        self.hass = hass

    async def export_data(self) -> dict:
        installed = []
        custom = []
        # 优先使用 HACS 内存数据（准确）
        if self.operator and self.operator.available:
            installed = self.operator.get_installed_list()
            all_repos = self.operator.get_all_repos_from_hacs()
            custom = [{"repository": r["full_name"], "category": r["category"]}
                      for r in all_repos if r.get("custom")]
        else:
            # Fallback: 文件读取
            installed = await self.data.get_installed_repositories()
            config = await self.data.get_config()
            custom = config.get("custom_repositories", [])
        return {"installed_repositories": installed, "custom_repositories": custom}
```

**`__init__.py` 同步修改**:
```python
# __init__.py L35

# 修复后
backup = BackupManager(hass, shared_data=shared_data, operator=operator)
```

```python
def get_installed_repositories(self) -> list[dict]:
    data = self._read_hacs_file("hacs.data")
    installed = []
    for repo in data.get("repositories", []):
        if isinstance(repo, dict):
            # 只返回已安装的（installed_version 非空）
            iv = repo.get("installed_version")
            if iv:  # truthy check: 过滤 "", None, missing key
                installed.append(repo)
    return installed
```

```python
def _resolve_ha_base_url(self) -> str:
```
