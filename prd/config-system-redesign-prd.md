# PRD: HACS Vision 配置体系重构

> 产品经理：许清楚
> 状态：草稿
> 日期：2026-06-18

## 1. 产品目标

重构 HACS Vision 的集成配置交互体系，使商店、集成管理、仓库管理三个视图职责清晰，按钮逻辑正确，操作完整。

## 2. 用户故事

### P0 — 商店卡片弹窗按钮正确
- **US-01**: 未安装的集成显示「安装」按钮
- **US-02**: 已安装未配置的集成显示「添加集成」按钮，点击打开 Config Flow
- **US-03**: 已配置且 supports_options=true 的集成显示「配置」按钮
- **US-04**: 已配置且 supports_reconfigure=true 的集成显示「重配置」按钮
- **US-05**: 已配置且 supported_subentry_types 非空的集成显示「添加设备/服务」按钮
- **US-06**: 已配置但无任何能力的集成不显示任何按钮
- **US-07**: 报错/重试状态的集成显示「查看日志」或「重试」按钮
- **US-08**: 用户禁用的集成显示「启用」按钮

### P1 — 集成管理页全能操作弹窗
- **US-09**: 点集成管理页的条目卡片，弹出全能操作弹窗
- **US-10**: 弹窗显示条目基本信息（域名、状态、来源）
- **US-11**: 弹窗按能力显示操作按钮网格（配置/重配/添加子条目/重命名/系统选项/重载/禁用/删除）
- **US-12**: 弹窗显示关联设备列表和实体数量
- **US-13**: 不支持的操作不显示按钮

### P2 — 配置弹窗 UX 优化
- **US-14**: 描述中的 URL 自动可点击
- **US-15**: 密码字段支持显示/隐藏切换
- **US-16**: 多步骤显示步骤进度（第 2/4 步）
- **US-17**: 提交时显示步骤进度文字
- **US-18**: 错误时保留取消按钮而不是全屏结果页

### P3 — 后端补充字段
- **US-19**: API 返回 supports_reconfigure
- **US-20**: API 返回 supports_remove_device
- **US-21**: API 返回 num_subentries
