## 二、代码审查摘要（19 个文件，48 个问题）

### 🔴 严重问题（6 个）

| # | 问题 | 位置 | 建议 |
|---|------|------|------|
| 1 | **内存泄漏**：requestAnimationFrame 循环中 `_lastAnimFrame` 未在 disconnectedCallback 中 cancel | `browse.js` 列表动画 | `disconnectedCallback` 中 `cancelAnimationFrame` |
| 2 | **SSH 连接池不释放**：`SSHClient` 未使用 `with` 或手动 close | `hacs_operator.py` | 改用 context manager 确保释放 |
| 3 | **Config Flow 不走后端代理**：直接调 `hass.callApi` 而非后端 api.py | `config-flow-dialog.js` | 统一经后端代理（api.js 已定义方法） |
| 4 | **「全部更新」语义错误**：按钮标"全部更新"实际只更新勾选的 | `updates.js` | 改名为"更新已选"或新增真正"全部更新"按钮 |
| 5 | **GitHub URL 搜索分页下失效**：粘贴 URL 时只在当前页过滤 | `browse.js` | 检测到 URL 时切到不分页模式 |
| 6 | **getRepoReleases 路径可能 404**：`_fetch('/repos/releases?...')` 拼接路径未验证 | `api.js` | 确认后端路由或改用统一 request() |

### 🟡 中等问题（19 个，节选）

| # | 问题 | 建议 |
|---|------|------|
| 7 | 更新后轮询永不结束（版本号前缀不匹配） | strip 'v' prefix 宽松匹配 |
| 8 | 进入更新页自动加载所有 changelog（N 个并行请求） | 改为懒加载 |
| 9 | `_cancelFlow` 与 `_close` 状态重置代码重复 | `_cancelFlow` 调 API 后只调 `_close` |
| 10 | 双重重置：dialog 和父组件都重置同一字段 | 只在一处重置 |
| 11 | 硬编码错误消息未走 i18n | 加 i18n key |
| 12 | API 层存在两套 fetch 实现 | 统一用 request() |
| 13 | 标签切换不保持子视图状态（滚动/搜索丢失） | 用 hidden 代替条件渲染 |
| 14 | `getRepoStatus` 没有 token 刷新逻辑 | 增加重试 |
| 15 | 配置流取消失败但静默忽略 | `console.warn` + 提示用户 |
| 16 | `close` 事件不携带结果数据 | 携带 result 类型 |

### 🟢 建议类（23 个，节选）

| # | 问题 | 建议 |
|---|------|------|
| 17 | 搜索热键 Ctrl+K 作用域过宽 | 检查焦点是否在面板内 |
| 18 | 本地存储 key 名分散各处 | 抽取到 constants.js |
| 19 | 收藏变更后不通知其他卡片刷新 | 用事件总线同步 |
| 20 | 添加自定义仓库后不更新分类计数 | 确保 API 返回完整 category_counts |

## 五、后期规划更新

### v2.1.0 — 短期（1-2 周）

**P0 优先（影响核心体验）**
- [ ] 修复「全部更新」按钮语义：改为「更新已选({n})」+ 新增真正的「全部更新」
- [ ] 修复搜索分页 URL 搜索失效，检测到 `github.com/` 时切到不分页模式
- [ ] 配置流改为走后端代理（统一 api.js 方法，记录审计日志）
- [ ] 修复标签切换丢失滚动位置（`hidden` 替代条件渲染）
- [ ] 统一三视图卡片渲染（抽取公用 repo-card 组件，属性控制差异展示）

**P1 重要优化**
- [ ] Tab 徽章绑定实际更新数量
- [ ] 更新页 changelog 改为懒加载
- [ ] 统一类别颜色常量，删除私有 `_getCategoryColor()`
- [ ] 统一 Spinner 尺寸规范
- [ ] 修复 Management 空状态 padding

### v2.2.0 — 中期（3-4 周）

**新功能**
- [ ] 收藏批量管理（查看所有收藏、移除）
- [ ] 安装历史/活动日志（Audit Log）
- [ ] 离线缓存最近浏览的仓库数据
- [ ] 一键全部更新（跳过选择直接更新所有待更新仓库）

**质量改进**
- [ ] 统一 fetch 实现，删除 api.js 中重复的 `_fetch()/_post()`
- [ ] 配置流外部认证后轮询完成状态
- [ ] 收藏状态全局同步（事件总线）
- [ ] 列表视图添加收藏按钮
- [ ] 错误处理全面 i18n 化

### v3.0 — 长期（1-2 月）

**重大功能**
- [ ] 自定义分类/标签系统（用户自己给仓库打标签）
- [ ] 仓库对比功能（同时查看多个仓库 README）
- [ ] 一键备份/恢复配置（集成列表、收藏、设置）
- [ ] 集成配置批量操作（批量删除/启用/禁用）
- [ ] HACS Vision 插件系统（第三方扩展能力）
- [ ] HACS Vision 统计仪表盘（安装量趋势、最受欢迎仓库等）

**架构改进**
- [ ] 前端状态管理（Redux 或 Zustand 替代分散的 state）
- [ ] 全面 TypeScript 化（提升类型安全）
- [ ] 单元测试 + E2E 测试覆盖（Playwright）
- [ ] 性能优化（虚拟滚动、大列表渲染优化）
- [ ] 浏览器历史记录支持（history.pushState）
